一、矩阵乘法三种方法的性能差异

从运行结果来看，np.dot、@ 和 np.matmul 的执行时间基本处于同一量级，差异并不明显。这是因为三者底层调用的都是同一个基础线性代数库（如 OpenBLAS 或 MKL），核心计算逻辑相同，因此性能上没有本质区别。
它们的主要差异在于使用场景和写法上：np.dot 是较早的函数，对一维和二维数组支持较好，但在处理高维数组时规则不够统一；np.matmul 是后来推出的标准化矩阵乘法函数，语义更清晰；@ 则是 Python 提供的运算符形式，实际上是 np.matmul 的语法糖，写法更简洁。
在实际开发中，推荐使用 @，既不影响性能，又能让代码更简洁易读。


二、金融数据分析中向量化的重要性

在金融数据处理中，数据量通常比较大，比如上千支股票、数百个交易日的数据。如果使用 Python 的循环逐个计算，不仅代码冗长，运行速度也会很慢，因为 Python 的循环本身效率较低。
向量化运算可以把循环操作交给 NumPy 底层用 C 语言完成，大幅提升计算效率。同时，向量化代码更加简洁，例如计算收益率只需一行 np.log(prices[1:] / prices[:-1])，表达式与数学公式几乎一致，可读性强，也便于后续修改和扩展。
因此，在金融数据分析中，向量化已经成为一种基本且必要的编程习惯。


三、用 NumPy 实现蒙特卡洛模拟股票价格

蒙特卡洛模拟的基本思路是：根据几何布朗运动模型，生成大量随机价格路径，用于估计未来价格的分布情况。
具体步骤大致为：设定初始价格、年化收益率和波动率等参数；生成多个独立的正态分布随机数，模拟每天的涨跌；利用累积求和构造随机游走路径；最后代入公式计算得到每条路径的价格序列。
NumPy 的优势在于可以同时生成成千上万条路径，所有计算一次性完成，速度远快于逐条循环生成。


四、什么情况下适合使用 order='F'（列优先）存储

NumPy 默认使用 C 风格（行优先）存储，即按行顺序在内存中排列数据。这种布局在按行访问时效率最高。
如果后续操作主要是按列访问数据，比如金融数据中每列代表一个时间点、每行代表一支股票，很多分析操作都是按时间列进行的，此时将数组改为 F 风格（列优先）存储，可以减少内存跳转，提高访问速度。
另外，如果调用的某些外部库是用 Fortran 编写的，它们默认按列存储，使用 F 连续可以避免数据格式转换的开销。
简单来说，主要按行操作就用 C 连续，主要按列操作就用 F 连续，根据数据的使用习惯选择即可。
